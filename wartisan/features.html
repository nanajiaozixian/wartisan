1. Easily create test case

  1. On the target page, invoke Wartisan UI to define your request. It will show up on the right side of your current page.
  2. The process to define a request is very easy
    • Select the action you want (update / new / delete)
    • Drag the related component to the action – the component information will show up in the action item
    • Describe the expected results.
  3. You can then save your request

2. Define your expectations

  1. Here the Update action is modified with expected content (show in Italic strings)
  2. You can also select a time range for this request.
  3. Once saved, the service will automatically monitor the expected content between the specified time range.

3. Check result

  1. When something not as expected, you will get an email notification.
  2. Following the link you will see the result from the monitoring service:
    • On which page
    • What is expected
    • What is actually found
  3. Result will be saved in our system for further reference. You can also view your corresponding request for details.

4. Case Management /Update

The details of a specific case will look like this: You can review and update your request, including:
  1. The subject of the request
  2. The time range
  3. How frequently you want to check
  4. Provide customized data through defining variables, the request will then expand to multiple pages.
  5. Add tags for further reference, one request could later join a request suite for further monitoring

5. Review cases

You can also review/check all the requests in our admin UI.
  1. Users can search the requests using Case ID, Subject, URL, etc.
  2. Users can create new request suits by selecting the checkbox in front of each case.
  3. Users can start an instant test on specific request
  4. Users can open a specific request to check details

6. Time-line

By using the smart time line, users can view/check a web page in a visible style.
  1. On February 1, the page may look like this
  2. On March 1, right circle changed to square
  3. On April 1, the title changed
  4. On May 1, left icon changed to circle
Each change reflects a request in system. You can easily find out what the page looked like in the past, and will look like in future.
